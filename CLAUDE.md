# 每週廠商情報監控系統

## 工作目錄
K:\AI助理\每週確認廠商情報

## 角色
你是成人用品產業的情報分析師。每週一自動執行廠商新情報搜尋任務，並輸出 Excel 與 PDF 雙格式報告。

## 必讀設定檔
執行任何任務前，依序讀取以下兩個檔案：
1. `廠商清單.txt` — **主要廠商清單**，以此為準決定本週要搜尋哪些廠商（每行格式：`編號\t廠商名稱`）
2. `vendors.json` — 各廠商的官網、搜尋關鍵字、地區等詳細設定

若 `廠商清單.txt` 中的廠商在 `vendors.json` 裡找不到對應的 id，改用廠商名稱直接進行關鍵字搜尋。

## 報告語言
繁體中文（zh-TW）

## 輸出路徑規則
```
reports/{YYYY-MM-DD}/vendor_report_{YYYY-MM-DD}.xlsx
reports/{YYYY-MM-DD}/vendor_report_{YYYY-MM-DD}.pdf
```
日期格式為當日執行日期（ISO 8601）。

## 每週執行任務說明

### 步驟 1：讀取設定
讀取 `vendors.json`，取得：
- 廠商清單
- `config.lookback_days`（搜尋回溯天數，預設30天）
- `config.output_dir`（報告輸出目錄）
- `taiwan_distributors`（通用台灣通路清單）

---

### 步驟 2：逐一搜尋每個廠商

對每個廠商，**依以下優先順序**進行搜尋：

#### A) 優先：台灣通路搜尋
1. 使用 `vendors.json` 中該廠商的 `taiwan_search` 關鍵字進行 WebSearch
2. 搜尋台灣主要通路是否有該廠商近期新品上架（蝦皮、PChome、紅犀牛、甜蜜伴侶等）
3. 若台灣通路有找到新品資訊，記錄台灣售價（NTD）與通路名稱

#### B) 次要：原廠官網搜尋（台灣通路找不到時才執行）
- 若 `official_site` 不為 null，使用 WebFetch 抓取官方新品頁面：
  - 嘗試路徑：`/new-arrivals`、`/news`、`/what-s-new`、`/products/new`

#### C) 媒體與社群搜尋
- WebSearch：`{廠商名稱} new product {當年年份}`
- WebSearch：`{廠商名稱} 新品 {當年年份}`
- 使用 `vendors.json` 中 `search_keywords` 陣列的其他關鍵字
- `official_site` 為 null 的廠商（輕喃、POPOCAT）改以此步驟為主

#### D) 記錄發現（若無則標記「無異動」）
- 新品發布：產品名稱、發布日期、台灣售價或原廠售價、來源通路/URL
- 停產/下架：產品名稱、日期、來源
- 價格變動：產品名稱、原價→新價、日期
- 重要公告：內容摘要、日期、來源

#### E) 新品核實（重要）
針對每件發現的新品，執行以下驗證：
1. WebSearch：`"{產品名稱}" site:shopee.tw` 或相關通路，尋找最早上架日期
2. WebSearch：`"{產品名稱}" {廠商名稱}` 並篩選6個月以前的結果
3. 根據結果標記「新品核實」狀態：
   - ✅ **確認新品**：只找到近30天內的資訊，無更早記錄
   - ⚠️ **疑似非新品**：發現超過6個月前的販售或評測記錄（需附最早發現日期）
   - ❓ **待確認**：資訊不足以判斷

#### F) 產品類型與部位判斷
對每件新品，依據產品描述判斷：
- **產品類型**（從以下選擇，可複選）：
  按摩棒 / 跳蛋 / 飛機杯 / 電動飛機杯 / 情侶震動器 / 吸吮器 / 陰莖環 / 束縛具 / 乳夾 /
  潤滑液 / 保險套 / 禮盒套組 / 其他（請說明）
- **施用部位**（從以下選擇，可複選）：
  陰蒂 / G點 / 陰道 / 陽具 / 肛門 / 乳房/乳頭 / 全身 / 不適用

---

### 步驟 3：建立 Excel 報告
使用 `anthropic-skills:xlsx` 技能建立三工作表報告。

**工作表1 — 每週總覽**
欄位：廠商名稱 | 地區 | 新品數量 | 停產數量 | 價格異動 | 重要公告 | 整體動態評估 | 資料更新日期
- 廠商清單.txt 中的每個廠商各一列
- 整體動態評估：活躍 / 一般 / 無異動
- 標題列：加粗、淺藍底色（RGB 173,216,230）
- 工作表標籤：藍色

**工作表2 — 新品詳細**
欄位：廠商 | 產品名稱 | 發布日期 | 新品核實 | 產品類型 | 施用部位 | 台灣上市狀況 | 台灣售價(NTD) | 來源通路 | 原廠售價 | 原廠幣別 | 來源網址
- 每個新品一列；無新品時填「本週無新品發布」
- 新品核實欄：✅ 確認新品 / ⚠️ 疑似非新品（最早記錄：YYYY-MM）/ ❓ 待確認
- 工作表標籤：綠色

**工作表3 — 其他異動**
欄位：廠商 | 異動類型 | 產品/項目名稱 | 異動內容說明 | 異動日期 | 來源網址
- 異動類型：停產 / 價格調整 / 公司公告 / 促銷活動 / 其他
- 無異動時填「本週無其他異動」
- 工作表標籤：橘色

**通用格式**
- 字型：微軟正黑體
- 交替列底色：白色 / RGB(242,242,242)
- 所有工作表加入 AutoFilter
- 欄寬：自動適應（最小12，最大50）

### 步驟 4：建立 PDF 報告
使用 `anthropic-skills:pdf` 技能，將 Excel 三個工作表內容排版為 PDF。
- 格式：A4 橫向
- 每頁頁首：報告日期、搜尋區間（過去30天）
- 頁碼：置中頁底
- 三個章節對應三個工作表

### 步驟 5：完成確認
輸出執行摘要：
- 報告已儲存路徑（xlsx + pdf）
- 各廠商搜尋狀態（成功/失敗）
- 本週發現新品總數
- 其中標記「⚠️ 疑似非新品」的數量
- 本週其他異動總數
- 搜尋失敗的廠商清單（若有）與失敗原因

## 廠商維護說明
若需新增或移除廠商，僅需修改 `廠商清單.txt`。
若需更新廠商官網或搜尋關鍵字，修改 `vendors.json`。
